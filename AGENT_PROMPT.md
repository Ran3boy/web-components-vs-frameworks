## Требования к Docker

Создай `Dockerfile`, который выполняет:

1. установку зависимостей;
2. сборку frontend-приложения;
3. запуск production-версии через nginx.

Примерная логика Dockerfile:

```Dockerfile
FROM node:20-alpine AS build

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=build /app/dist /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

Также создай `docker-compose.yml`:

```yaml
services:
  app:
    build: .
    ports:
      - "8080:80"
```

## GitHub Actions

Создай файл:

```text
.github/workflows/ci-cd.yml
```

Pipeline должен выполнять:

- загрузку репозитория;
- установку Node.js;
- установку зависимостей;
- запуск тестов;
- сборку проекта;
- публикацию приложения на GitHub Pages.

Пример структуры pipeline:

```yaml
name: CI/CD

on:
  push:
    branches:
      - main

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm test -- --run
        continue-on-error: true

      - name: Build project
        run: npm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: dist

  deploy:
    runs-on: ubuntu-latest
    needs: build

    environment:
      name: github-pages

    steps:
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4
```

## GitLab CI/CD

Если используется GitLab, создай файл:

```text
.gitlab-ci.yml
```

Pipeline должен выполнять:

- установку зависимостей;
- тестирование;
- сборку;
- публикацию на GitLab Pages.

Пример:

```yaml
image: node:20-alpine

stages:
  - test
  - build
  - deploy

test:
  stage: test
  script:
    - npm ci
    - npm test -- --run
  allow_failure: true

build:
  stage: build
  script:
    - npm ci
    - npm run build
  artifacts:
    paths:
      - dist

pages:
  stage: deploy
  script:
    - npm ci
    - npm run build
    - rm -rf public
    - mv dist public
  artifacts:
    paths:
      - public
  only:
    - main
```

## Требования для публикации на GitHub Pages

Если проект публикуется на GitHub Pages, учти, что приложение может быть размещено не в корне домена, а по адресу вида:

```text
https://USERNAME.github.io/web-components-vs-frameworks/
```

Поэтому настрой `base` в `vite.config.ts`:

```ts
export default defineConfig({
  base: "/web-components-vs-frameworks/",
  plugins: [react()]
})
```

Если проект запускается локально или публикуется на собственном домене, объясни в README.md, как изменить значение `base`.
