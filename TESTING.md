# Процесс тестирования

В рамках данного проекта тестирование настроено для проверки работоспособности CI/CD конвейера и базовой логики приложения.

## Реализованные тесты
- **Unit-тесты:** Реализованы с использованием `vitest`. Проверяется доступность тестового окружения и базовые функции.
- **Сборка (Build test):** Проверяется успешность транспиляции TypeScript-кода и сборки Vite-приложения.
- **Контейнеризация:** Проверяется успешная сборка многоэтапного `Dockerfile`.

## Запуск тестов локально

Выполните команду:
```bash
npm test
```
Тесты будут запущены в режиме `watch`. Для однократного прогона в CI используется:
```bash
npm test -- --run
```

## Результаты для отчета ВКР
Успешным результатом считается выполнение команды `npm run test` без ошибок, а также отсутствие сбоев на шагах установки зависимостей (`npm ci`) и сборки (`npm run build`) в логах GitHub Actions. 
Скриншоты успешного выполнения (зеленые галочки) из вкладки `Actions` в GitHub могут быть приложены к отчету.

## Дополнительные аспекты проверки
1. **Как проверяется корректность отображения компонентов:** Визуальное QA-тестирование на разных устройствах через интерактивный дашборд стенда.
2. **Как проверяется сборка проекта:** Локально через команду `npm run build` и автоматически в CI-конвейере (шаг Build).
3. **Как проверяется Docker-образ:** Локальный запуск `docker compose up --build` с проверкой доступности Nginx на порту 8081.
4. **Как проверяется CI/CD:** Автоматическими логами GitHub Actions при коммите в ветку `main`.
5. **Как интерпретировать метрики дашборда:** Чем выше Итоговая Оценка, тем сбалансированнее подход. Размер и Время (меньше = лучше), Переиспользуемость и Поддержка (больше = лучше).
